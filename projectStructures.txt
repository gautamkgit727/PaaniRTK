📌 Best Practices

Feature-first structure → Each feature manages its own slice, API, components, and pages.

RTK Query → Use it for API requests, caching, and invalidation (instead of Axios everywhere).

Shared components in /components → Button, Modal, Table, Loader.

Layouts → Keep Admin/Public separated.

Utils/hooks → For common logic (useAuth, useFetch, debounce).

Absolute imports → Configure Vite alias (@/features/auth/...) in vite.config.js.

Testing ready → Write Jest/RTL tests inside __tests__ or next to components.
